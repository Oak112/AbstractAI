# 从冗长混乱的上下文到一套可执行的产品文档

**项目地址：** https://abstractai.ai-builders.space/

---

## 问题

我在开发新项目时，习惯先和 ChatGPT 聊很多轮，形成一个很长的上下文。然后把它直接丢给 Cursor / Augment Code / Claude Code，让它根据这些对话去实现。

这个工作流有两个问题：

1. **上下文太长，AI 容易跑偏。** 对话中可能对同一个点反复讨论、推翻、修改，AI 很难判断哪个是最终结论，容易抓不住重点。

2. **从 idea 直接到代码，跳过了太多环节。** 对于非技术背景的人，让 AI 从一个模糊的想法直接写出完整系统，成功率很低。因为中间缺了 PRD、系统设计、架构设计、接口定义这些关键步骤。

---

## 解决方案

在"对话上下文"和"AI 写代码"之间，加一个中间环节：**把上下文编译成一套完整的产品文档**。

输入：任意长度的文本（和 AI 的对话、会议录音转录、项目描述……）

输出：11 份结构化文档，包括：
- 产品总览（给所有人看）
- 功能与规则说明（给产品和测试看）
- 技术实现与架构（给工程师看）
- 术语表（给新手看）
- 任务拆分与验收（给执行用）
- 决策日志、边界情况、原话摘录、追溯映射……

这些文档：
- **新手友好**：专业术语都有解释，非技术人员也能看懂
- **工程可执行**：工程师或 AI 可以直接照着做
- **证据可追溯**：每个决策都标注了来源，引用原话

---

## 效果

这个项目本身是用 Augment Code 写的，prompt 是 GPT-5-pro 帮我写的。从开发到优化到修 bug，**一共花了 20 美金**。

用同样的方法，我开发了下一个项目——一个复杂得多的完整系统。因为 AI 在执行前已经通过文档明确了一切，照着做就行，不用在模糊的需求里来回改、来回问。**那个项目的花费也是 20 美金**。
